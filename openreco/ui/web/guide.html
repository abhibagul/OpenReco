1 · Overview

OpenReco turns a set of overlapping photos into 3D and mapping products: a sparse/dense point cloud, a 3D mesh with texture, a digital elevation model (DSM), an orthomosaic, contours, and vegetation indices — which you can measure, annotate and export. It is a clean-room, open project built only on permissively-licensed software (see Help ▸ Credits & licenses).

The pipeline is a chain of stages (layers) shown in the Workspace tree. Each stage caches its result, so re-running only recomputes what changed.

2 · Getting started

Launch: double-click openreco.exe — the app opens directly (no commands needed). A small console window opens alongside it showing live logs; leave it open while you work.

Check your machine: Help ▸ System / compute info (or openreco doctor) shows whether your GPU, the CUDA dense engine and all components are available.

GPU: an NVIDIA GPU + driver gives fast dense reconstruction. Without one, OpenReco still runs on the CPU (slower, and dense detail is limited).

3 · Projects & photos

A project is a folder that holds your pipeline and results.

  1. File ▸ Open / new project → Browse to a folder and click Use this folder. An existing project opens; an empty folder becomes a new project.
  2. Add photos: right-click a chunk (or use the Workflow menu) ▸ Add Photos → browse to your image folder, tick the photos, and add them.

Right-click any photo thumbnail for: Open / preview, Look through (move the 3D camera to that shot), Show info (EXIF/GPS), Enable/Disable, Move to another chunk, Estimate image quality (sharpness), and Remove.

Use chunks to keep separate flights/areas apart, then Merge Chunks later. Disable blurry photos (their sharpness shows on the thumbnail after Estimate quality).

4 · The processing workflow

Open the Workflow menu and run these in order (each adds a layer wired to the previous):

StepWhat it produces
Align PhotosCamera positions + a sparse tie-point cloud (Structure-from-Motion).
GeoreferencePlaces the model in real-world coordinates using photo GPS or GCPs.
Build Dense CloudA detailed point cloud (GPU PatchMatch, or CPU fallback).
Build ModelA 3D mesh surface from the dense cloud.
Build TexturePhoto-realistic texture baked onto the mesh.
Build DEM / OrthomosaicTop-down elevation raster & stitched map image.
Vegetation Indices / ContoursNDVI/ExG maps & contour lines.

Quality preset: the toolbar dropdown (Low / Medium / High / Ultra) sets sensible parameters across all stages in one click. Start Low for a quick preview.

Run: press Run in the toolbar. OpenReco first checks the wiring and lists any problems; then it processes, streaming progress to the Console. Re-running is cheap — cached stages are skipped. Right-click a layer to Run just that stage or Run up to here.

On a 4 GB GPU, Low/Medium are the reliable dense settings. If dense fails with a CUDA error it automatically retries at a smaller size before falling back.

5 · Coordinate systems & GCPs

To get real-world coordinates, the project needs a CRS (coordinate reference system) and a georeference source:

  • GPS: if your photos are GPS-tagged (most drones), add a Georeference step wired to the chunk's photos — it auto-picks a UTM zone or uses the CRS you set (toolbar ▸ CRS).
  • GCPs (Ground Control Points): for survey accuracy, open Reference ▸ Markers, add markers, click their position in several photos, enter known coordinates, then Use these GCPs.
If a model comes out un-georeferenced (the map/volumes are off or local), the pre-run check warns you: the Georeference step has no GPS or GCP source. Wire the chunk's photos to it, or add GCPs.

6 · Viewing results

The viewport has four tabs (top-left): Model (3D), Ortho (2D map image), Map (web map), Photo.

  • Orbit drag; pan right-drag/middle-drag; zoom wheel.
  • Navigation cube (top-right): click a face/edge/corner to snap the view.
  • Point-cloud bar (bottom): point size, color by RGB or Elevation (ramp), and EDL depth shading for clarity.
  • Toggle layers with the eye icon in the Workspace tree; double-click a layer to open it in the right view.

7 · Measurements & annotations

Use the measure tools in the toolbar (they work in the 3D, Ortho and Map views):

ToolHow
DistanceClick points along a path; double-click to finish.
AreaOutline a polygon; double-click to finish.
Volume (stockpile)Outline a footprint; double-click. Pick the base (best-fit plane / lowest / mean) — reports cut/fill/net m³.
ProfileClick points along a line; an elevation cross-section chart appears.
NoteClick a point and type a label.

All measurements appear in the floating Measurements panel (rename, show/hide, delete, click to focus). They are saved with the project and shown in 3D, on the ortho and on the map. Export them from the panel as GeoJSON / DXF / CSV; a profile exports a per-station CSV.

Set display units (metre / feet) and decimal precision in File ▸ Preferences ▸ Measurements.

8 · Vegetation indices

Workflow ▸ Vegetation Indices (needs an Orthomosaic) computes plant-health maps:

  • RGB (ExG · VARI · GLI) — works on ordinary RGB drone photos.
  • NDVI / GNDVI — require a near-infrared (NIR) band, i.e. a multispectral camera; set the NIR band # to match. On RGB-only data these are skipped.

Each index is written as a GeoTIFF plus a colorized (red→green) image you can view and export.

9 · Cleaning & editing

  • Clean Point Cloud / Clean Mesh (Workflow) removes noise / small floating islands.
  • 3D edit tools (toolbar): box or lasso select, visible-only mode, then delete. Save edits as a new layer (non-destructive — the original is kept).

10 · Export, report & jobs

  • Export a layer: select it → the Properties pane ▸ Export → choose a format (PLY/LAS/LAZ, OBJ/GLB/STL, GeoTIFF, GeoJSON/DXF, …).
  • Processing report: File ▸ Processing report (PDF) — a summary with stats, GCP accuracy and your measurements.
  • Jobs panel (bottom dock): the history of every run — status, time, duration; click a row to open that run's report.

11 · Preferences & compute

File ▸ Preferences: measurement units & precision, default view, navigation speed, console size, and a Compute panel showing your GPU / CUDA-COLMAP / dependency status (the same as openreco doctor).

12 · Troubleshooting

SymptomFix
Dense fails with a CUDA "misaligned address" errorIt auto-retries smaller; if it still fails, set the dense layer's quality = Low and re-run. Common on 4 GB GPUs at high quality.
"GPU: none detected" / dense is slowInstall/update your NVIDIA driver. Without a GPU, OpenReco uses the CPU (sparse cloud).
Map or 3D view is blankThe 3D/map libraries load from the internet — connect to the network, or ask for an offline build.
Model is "local" / not georeferencedAdd a Georeference step wired to the chunk's photos (for GPS), or set GCPs. The pre-run check warns about this.
Windows blocks the exe on first runSmartScreen → More info ▸ Run anyway (the build is unsigned).
NDVI is empty / skippedNDVI needs a NIR band (multispectral camera). On RGB photos use ExG / VARI / GLI instead.

13 · Command line (optional)

Power users can drive everything headless:

openreco doctor                         # GPU / COLMAP / dependency status
openreco init myproj --images C:\photos --crs EPSG:32613
openreco run  myproj                    # process headless
openreco ui   myproj                    # open the app on a project
openreco report myproj                  # locate the latest report
